Picmv ver.1.60


●Picmv とは

・デジタルカメラで撮影した画像を任意のディレクトリにコピーするフリーソフトウェアです。
・複数のデジタルカメラとメモリカードリーダを持っている場合に便利です。
・Exifヘッダの撮影日時情報を元にファイル名に命名します。命名規則はユーザが定義することができます。
・JPGファイル以外も任意の拡張子のファイルを対象にできます。
・メモリカードを自動認識することができます。手動で設定することもできます。
・元ディレクトリは複数指定することができ、存在しなくてもエラーになりません。これは
  マルチメモリカードリーダを使ったときに便利に使えるようにするためです。
・コピー終了後にメモリカードを自動的にイジェクト(取り出し)することができます。
・コピー先ディレクトリ名も撮影日時情報から命名することができます。
・サムネイル(縮小画像)を出力することができます。
・縦画像属性により画像を回転して出力することができます
・コピー中は OpenGL を使ったグラフィカルな進捗表示を行います。特にメモリカードを自動認識する
  設定にしている場合は、メディアの挿入とイジェクトがアニメーションで表現されます。
  また、ヒストグラムを表示することもできます。
・コピー中、いつでも中止することができます。

大切な写真ファイルを失わないように、お使いのカメラ、PC環境で十分にテストをしてください。
全てのカメラ・メモリカードリーダでのテストができません。
何が起きても責任は取れません。



●動作環境

Windows 10



●ZIP 版のインストール方法

適当なディレクトリを作成し、アーカイブを展開します。



●ZIP 版のアンインストール方法

インストールしたディレクトリを丸ごと削除します。Picmv.exe 自体がレジストリを利用することはありません。



●インストーラ版のアンインストール方法

Windows の標準的な方法でアンインストールします。
コントロールパネル→アプリケーションの追加と削除→プログラムの変更と削除→Picmv→変更と削除を
クリックします。



●使い方

1. Picmv.exe を起動します
2. ファイル入力元、出力先、命名規則などを適切に設定し、実行ボタンを押します
3. コピーされます

または、

1. Picmv.exe を起動します
2. ファイル入力元、出力先、命名規則などを適切に設定し、ウィンドウにファイルやフォルダをドロップします
3. コピーされます



●ファイル・ディレクトリの命名法について

%Y 4桁の年号
%y 2桁の年号
%m 月
%d 日
%H 時
%M 分
%S 秒
%w 週(英語表記)
%W 週(日本語表記)
%k メーカー名
%l モデル名
%g 元号による年号

例: %y%m%d_%H%M%S → 050315_221202

例えば、撮影日時が2005年2月14日12:34:59の場合、
%Y=2005,%y=05,%m=02,%d=14,%H=12,%M=34,%S=59に置換されます。
EXIFヘッダがない場合や JPEG ファイル以外ではタイムスタンプを利用します。

出力先ディレクトリにも撮影情報からの % 指定は有効です。Picmv は指定したディレクトリが
存在しないときは確認なしに作成します。
上記の % 表記に「a」を挿入した、%aY、%ay、%am、%ad、%aH、%aM、%aS、%aw、%aW では実行時刻に置換されます。
実行時刻とは実行ボタンを押した瞬間の時刻です。ディレクトリの命名に使用すると
月日をまたがるイベントの取り込みに便利かもしれません。

%% は % 文字自体を示します。

ファイル命名法に拡張子は記述できません。

%o は元ファイル名・ディレクトリ名に置換されます。

ファイル名が重複する場合の処理は、シリアルナンバを付加するか、コピーしないか、
上書きするかをあらかじめ選択します。



●注意事項

・メモリカードの自動認識は、リムーバブルディスクでなおかつルートに DCIM というディレクトリがあるとそれと認識されます。
・画像処理系の機能はJPEGにのみ有効です
・サムネイルは常にバイリニア補間とアンシャープマスク処理を行っています。
・イジェクト機能は、エクスプローラーでいうところの「リムーバブルディスクの取り出し」です。「ハードウェアの取り外し」
  「ハードウェアの停止」ではありません。
・ドラッグアンドドロップは起動している状態でのみ受け付けます。エクスプローラーから Picmv.exe にドロップしても起動するだけです。
・元ファイルがリードオンリー属性の場合、コピー後のファイルは常にリードオンリー属性を解除します。
・元ファイルのタイムスタンプは、常にコピー後のファイルにそのままコピーされます。
・JPEG ファイル以外のプレビューはできません。
・PREVIEW という名前のフォルダのファイルはコピーしません。
・オプションと設定は「設定保存」ボタンでインストールしたディレクトリの PicMove.ini に保存されます。
・デフォルトの状態に戻すには PicMove.ini を削除してください。
・上書き設定は非推奨としました。
  最悪でファイル消失の危険があります。
  なぜなら、同時刻 EXIF 情報をもった写真や、EXIF なしで同時刻タイムスタンプをもった写真を
  一括コピーする場合に、同一ファイル名になっていまったファイル群が、
  上書きで最後のコピーだけが生き残る場合があるからです。
  このとき、もしも「元ファイルを削除」にチェックをいれていた場合、ファイルを失うことになります



●シリアルナンバについて

Picmv ではファイル名にシリアルナンバを含ませることができます。この機能は
ファイル名が重複したときに上書きを回避するためだけに存在しています。

ファイルの命名においては、一貫性があり永続的に使用可能な名前をつけるべきだと思います。
よって %y%m%d_%H%M%S と秒単位までの名前を命名することをお勧めします。
%y%m%d などとして積極的にシリアルナンバを付加するのは、
一貫性のない名前になりやすく全くお勧めできません。したがって秒2コマ以上撮影することを
普段からされている方に Picmv をお勧めすることはできません。




●オプション

・メモリカード自動検出-検出間隔
    「メモリカードを自動認識させる」チェックボックスがオンになっている場合のみ有効です。
    リムーバブルディスクのルートディレクトリにDCIMという名前のディレクトリが存在するかどうかを確認する間隔です。
    1秒から5秒まで指定できます。あまり短くするとパフォーマンスが低下する可能性がありますし、
    長くすると応答性が下がりユーザビリティが下がりますので、適当な位置にセットする必要があります。

・コピー処理進捗
    コピー処理を行っているときの進捗表示の負荷をコントロールします。
    ここでの設定は Picmv の動作結果そのものには影響を与えませんが、
    楽しい経過表示をさせたいときは 工夫して設定するとよいでしょう。

・入力-入力ファイル拡張子
    「JPG,JPEG,AVI」などと、入力させたいファイルの拡張子を,(カンマ)で区切って記述します。
    JPG は常に存在し消去することができません

・出力-JPEG生成時の品質設定
    Picmv が JPEG ファイルを生成するときの品質を設定します。
    JPEG ファイルは通常コピーされて 出力されるので、この場合は品質の設定は無関係です。画像処理やサムネイルにだけ関係します

・ファイル名重複時に追加するシリアルナンバ書式
    「ファイル名重複時の処理」で「シリアルナンバを付加する」設定になっている場合にのみ有効です。
     区切り文字列と、桁数を指定します。シリアルナンバは十進数で表現されます。

・拡張子の大文字・小文字変換設定
    入力ファイル名の拡張子を大文字小文字変換するか、なにも変換しないかを選択します。

・スマートコピー
    重複時上書き設定でファイル内容同一ならコピーをスキップします
    重複時シリアルナンバ付加設定でファイル内容同一のものを発見したらコピーをスキップします
    ファイル内容の同一性の確認の処理が重いことがあるのがデメリットです
    スマートコピーは安全かつ便利なので作者はこれを推奨しています





●コマンドラインオプション

オプションは設定が保存されている Picmove.ini の内容の上に設定されます。
オプションは自動的に保存されません。
オプションは大文字小文字を区別しません。

書式: Picmv <indir> <options...>

オプション:

-confirm [on|off]
    処理内容の事前確認を指定します

-autoquit [on|off]
    実行後に終了することを指定します

-autodetect [on|off]
    メモリカードの自動認識を指定します

-deleteorg [on|off]
     元ファイルの削除を指定します

-indir "ファイル入力元ディレクトリ群,..."
    入力元ディレクトリをセットします

-outdir "出力先ディレクトリ命名法"
    出力先ディレクトリ命名法をセットします

-outfile "ファイル命名法"
    ファイル命名法をセットします

-makethumbnails [on|off]
    サムネイルの作成を指示します

-thumbnailwidth [width]
    サムネイルの幅を指定します

-thumbnailoutdir "サムネイル出力先ディレクトリ命名法"
    サムネイル出力先ディレクトリ命名法をセットします

-thumbnailoutfile "サムネイルファイル命名法"
    サムネイルファイル命名法をセットします

-ow_addsn
    ファイル名重複時にシリアルナンバの付加を指定します

-ow_snchar "シリアルナンバとの区切り文字"

-ow_sndigits [桁数]

-ow_nocopy
    ファイル名重複時にコピーしないことを指定します

-smartcopy [on|off]
    スマートコピーを指定します

-rotatebyexif [on|off]
    オリエンテーション属性より回転します

-accept  [拡張子,...]
    受け付ける拡張子を指定します

-addsn [on|off]
    常にシリアルナンバをつけます

-autoexec
    自動的にコピー処理の実行を開始します
